## Структура проекта

Проект выполнен в одном Maven-проекте с разделением на пакеты.

- `src/main/java` — основной код
- `src/test/java` — JUnit-тесты
- `src/docs` (или `docs`) — HTML-документация (Javadoc)

Пакеты:
- **ru.semeshko.annotations** — аннотации
- **ru.semeshko.examples** — примеры классов, отмеченных аннотациями
- **ru.semeshko.handlers** — обработчики аннотаций (рефлексия)
- **ru.semeshko.main6** (или ваш пакет main) — точка входа `main`
- **ru.semeshko.tests** — тесты JUnit (в `src/test/java`)

## Задание 1  
**Аннотации (1–6)**

В этой части реализованы собственные аннотации и показана их работа через обработчики.

---

### Задание 1.1 — `@Invoke`

**Условие**  
Создать аннотацию `@Invoke`, которой помечаются методы, которые нужно вызывать автоматически.

**Алгоритм**
1. Создать аннотацию `@Invoke` с `RetentionPolicy.RUNTIME`.
2. Создать обработчик `InvokeHandler`.
3. В обработчике пройтись по методам класса через Reflection.
4. Вызвать методы, помеченные `@Invoke`, которые не имеют параметров.

**Демонстрация**
- В `main` создаётся объект `InvokeExample`
- Обработчик вызывает все методы с `@Invoke`
- Выводится количество вызванных методов

---

### Задание 1.2 — `@Default`

**Условие**  
Создать аннотацию `@Default`, которая хранит класс «по умолчанию».

**Алгоритм**
1. Создать аннотацию `@Default(Class<?> value)`.
2. Реализовать `DefaultHandler`, который считывает аннотацию с класса/поля.
3. В `main` вывести класс по умолчанию для примера.

**Демонстрация**
- В `main` читается `@Default` у класса `DefaultExample`.

---

### Задание 1.3 — `@ToString`

**Условие**  
Создать аннотацию `@ToString`, которая управляет включением полей в строковое представление.

**Алгоритм**
1. Создать `@ToString` и режим `Mode { YES, NO }`.
2. Реализовать `ToStringHandler`, который:
   - берёт все поля через Reflection
   - исключает поля с `@ToString(NO)`
3. Собирает строку вида: `ClassName{field=value, ...}`

**Демонстрация**
- В `main` создаётся объект `ToStringExample`
- Сравнивается обычный `toString()` и результат обработчика

---

### Задание 1.4 — `@Validate`

**Условие**  
Создать аннотацию `@Validate`, содержащую массив классов для проверки.

**Алгоритм**
1. Создать `@Validate(Class<?>[] value)`.
2. Реализовать `ValidateHandler`, который получает массив классов.
3. Для варианта 9: если массив пустой — выбрасывать `IllegalArgumentException`.

**Демонстрация**
- В `main` печатаются классы из `@Validate`.
- В тестах проверяется выброс исключения на пустом массиве.

---

### Задание 1.5 — `@Two`

**Условие**  
Создать аннотацию `@Two` с двумя обязательными параметрами: `first` и `second`.

**Алгоритм**
1. Создать `@Two(first, second)`.
2. Реализовать `TwoHandler`, который:
   - читает значения
   - проверяет корректность:
     - `first` не пустой
     - `second` не отрицательный
   - иначе выбрасывает `IllegalArgumentException` (вариант 9)

**Демонстрация**
- В `main` выводятся значения `@Two`
- В тестах проверяется, что при некорректных значениях выбрасывается исключение.

---

### Задание 1.6 — `@Cache`

**Условие**  
Создать аннотацию `@Cache`, содержащую массив строк (областей кеша).

**Алгоритм**
1. Создать `@Cache(String[] value)`.
2. Реализовать `CacheHandler`, который считывает области и печатает их.

**Демонстрация**
- В `main` печатается список кешируемых областей.

## HTML-документация (Javadoc)

Документация генерируется через IntelliJ:

**Tools → Generate JavaDoc...**

Рекомендуемые аргументы:
-encoding UTF-8 -charset UTF-8 -docencoding UTF-8
Папка вывода: `src/docs` (или `docs`)

Результат: `index.html`, `allclasses-index.html`, `allpackages-index.html`, папка `index-files/` и т.д.
